Variable Scope:
A Handy Primer

Sunday, January 11, 2015

Our variables in Ruby can be referenced in many different locations in a program. The range in which a variable can be referenced is called its scope. Ruby has five different variable scopes, each with its own sigil which is attached to the variable.

Global variables

Global variables can be accessed from anywhere and start with $, for example: $global_value. Uninitialized global variables have the value nil. Using global variables is generally discouraged, for they can easily lead to errors and should only be used when there is a good reason.

Constants

Constants can be accessed from iniside a class if defined inside a class, otherwise they can be accessed globally. A constant begins with an uppercase letter and is generally written in all caps, for example: CONSTANT. Constants must be initialized with a value and making a change to an already intialized constant will produce a warning. Constants are used for values that (ideally) will not change in the program.

Class variables

Class variables are valid for all instances of the class they are defined within and start with @@, for example: @@class_variable. Class variables must be initialized before they can be used in a method and referencing an unitialized class variable will produce an error. Class variables are used when you want many instances of a class to share a certain value.

Instance variables

Instance variables are valid for a single instance of a class and start with @, for example: @instance_variable. Uninitialized instance variables have the value nil. Instance variables are used when you want instances of a class to have different vaues for a certain variable.

Local variables

Local variables are only valid within a local area of code, such as a method or class. Local variables have no sigil and begin with a lowercase letter or underscore, for example: local_variable. An uninitialized local variable is treated as a call to a method with no arguments.
