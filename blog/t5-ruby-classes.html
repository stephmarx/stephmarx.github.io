The Classiest Blog Entry Yet

Sunday, January 4th, 2015

We can use classes to create our own, brand-spankin-new types objects within Ruby. When we initialize the class, we determine what properites the objects in that class will have and we can even write our own class methods to determine some things we can do with those objects.

Let's say I want to store and change information about some Dungeons and Dragons characters in a game I'm running. I can do this by creating my own class called Character:

class Character
def initialize(race, class, level, name)
@race = race
@class = class
@level = level
@name = name
end

Here, I've created my class and defined the method for initializing this class. Specifically, when we create a new character, we need to give its race, class, level, and name, like so:

Valdrox = Character.new(orc, fighter, 17, Valdrox)

So far, so good. As part of initializing this class, I created four instance variables (race, class, level, and name) that we can use across other class methods. We've used the Character class to store some information about Valdrox in these instance variables. What if we want to change these instance variables?

attr_accessor :race :class :level :name

When I define the class, I can declare which variables associated with the class can be read and written to. Here, I've chosen to make these variables both readable and writable with attr_accessor. So, what else can I do, now that I've created a character?

def level_up
@level +=
puts "Congrats, you've gained a level!"
end

def who_am_i
puts "#{name} is a level #{level} character and is a #{race} #{class}."
end

Here, I've defined a class method that levels up the character by changing the @level value. I've also created a method that returns the informaiton about the character we've given so far. Let's see how all these methods look together as part of the class declaration:

class Character
attr_accessor :race :class :level :name

def initialize(race, class, level, name)
@race = race
@class = class
@level = level
@name = name
end

def level_up
@level +=
puts "Congrats, you've gained a level!"
end

def who_am_i
puts "#{name} is a level #{level} character and is a #{race} #{class}."
end

So, if we enter the following:

Arwyn = Character.new(Elf, Druid, 3, Arwyn)
puts Arwyn.level_up
puts Arwyn.who_am_i

Our code would return:

"Congrats, you've gained a level!"
"Arwyn is a level 4 character and is a Elf Druid."
