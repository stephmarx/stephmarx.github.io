Loops in Javascript vs Ruby

Sunday, January 18, 2015

Let's take a look at a simple for loop in Ruby:

      for x in 1..5
        puts "#{x} is a number."
      end

This simple loop will print out a statement about the numbers one through five and then end. What would a for loop that outputs the same text look like in JavaScript?

      for (x=1; x<6; x++) {
        console.log(x + " is a number.")
      }

Here, we are explicitly declaring a variable x and setting it equal to 1. Then, we say that the loop should continue as long as x<6. After each time we execute the block of code in the loop, we increment x by one. In Ruby, we don't have to declare the variable at any point to use it as a counter in this loop. Let's take a look at while loops next, starting with Ruby:

      x = 1
      while x < 6 do
        puts "#{x} is a number"
        x += 1
      end

Here, we do explicitly declare the variable we use as our counter before we enter the loop. Then, we say that while the counter is less than 6, we should ouput our phrase and then increment the counter. The loop will repeat while our condition is true. What about in JavaScript?

      var x = 1;
      while (x < 6) {
        console.log(x + " is a number")
        x++
      }

As in our Ruby while loop, we declare our counter variable and its initial value before the loop. Then, we say that while the counter is less than 6, print out our phrase and increment the counter. The structure of the while loop is remarkably similar across both languages, unlike the for loop. Aside from the do...while loop in JavaScript, which is very similar to the while loop, that's pretty much it for loops in JavaScript, suprisingly! In Ruby, we are also able to use each, until, and .times to iterate code.
